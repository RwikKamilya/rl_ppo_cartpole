# Assignment 4: PPO-Clipped on CartPole-v1

Proximal Policy Optimisation (PPO-clip) implementation from scratch in PyTorch.  
Research question: *Does PPO-clipped retain the sample efficiency of A2C while improving policy-update stability through a clipped surrogate objective?*

---

## Quick Start

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Smoke test (finishes in ~30 seconds)

# 3. Full run вЂ“ one seed (в‰€ 5вЂ“10 min on CPU)
```

## Output Format

**Train CSV** (`results/<exp>/seed_<n>/train.csv`):
```
episode, step, return
```

**Eval CSV** (`results/<exp>/seed_<n>/eval.csv`):
```
step, mean_return, std_return
```
Deterministic evaluation (argmax policy) every 10,000 steps, 20 episodes each.

**Update log** (`results/<exp>/seed_<n>/update_log.csv`):
```
update, step, policy_loss, value_loss, entropy, approx_kl, clip_fraction
```
